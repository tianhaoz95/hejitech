ARCHITECTURE

Open Reference Workspace

This repository acts as a unified root workspace containing our primary landing page under home/ and our core product codebases linked dynamically as git submodules under projects/.

By auditing or checking out our submodules, developers can inspect implementation details, architectural conventions, and deploy templates that we use across all of our systems.

workspace ~ hejitech
$ git submodule status

a5f2b8e projects/layerforge (heads/main)

PRINCIPLES

How we design and engineer.

01

On-Device Intelligence

Wherever possible, we move model execution to the client device. This guarantees maximum privacy, zero backend latency, offline capability, and reduces infrastructure costs.

02

Socratic Guidance

We avoid AI interfaces that simply spit out copy-paste code. Our products focus on helping developers think through problems, providing Socratic cues and tests that build skill.

03

Visual & Code Aesthetics

We believe that clean, beautiful user interfaces reflect clean, beautiful codebases. We maintain high standards of styling, typography, and structure in everything we ship.

QUESTIONS

Frequently Asked Questions

What is Heji Technology LLC?

Heji Technology LLC is an AI-native product laboratory and infrastructure builder. We specialize in creating high-performance developer tools, investment platforms, and local-first AI applications where artificial intelligence operates as a first-class citizen.

How do I inspect and run the submodules in ./projects?

Each project under the projects/ directory is linked as a git submodule. You can pull them down locally by running: 

git submodule update --init --recursive

Once updated, follow the setup and development guidelines in the respective README.md files inside those directories.

Are Heji Technology projects open source?

We operate under an "open-reference" model. Many of our core libraries and configuration files are publicly readable for architectural audits and educational purposes, while certain enterprise apps remain closed-source but referenceable.
